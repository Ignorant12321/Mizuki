### 服务器费用

服务器购买【配置费】大致分为以下几类：包年包月、按量付费、抢占式实例。【本表格由Gemini Pro生成】

| **计费类型**                   | **核心逻辑**                                      | **优点**                                                     | **缺点/风险**                                                | **推荐场景**                                                 |
| ------------------------------ | ------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **包年包月** (Subscription)    | **先买票，后上车** 一次性支付一个月或一年的费用。 | 1. **价格优惠**：平均每天的成本比“按量付费”低很多。 2. **稳定**：资源完全归你，不会被收回。 3. **备案首选** | 1. **灵活性低**：买了就得用，中途退款比较麻烦。 2. **初期投入高**：需要一次性付一笔钱。 | **长期运行的网站**（如公司官网、个人博客）、核心数据库、需要备案的业务。 |
| **按量付费** (Pay-As-You-Go)   | **打车模式** 用一小时扣一小时的钱，随时可以删除。 | 1. **极度灵活**：随时开通，随时释放，没有资金压力。 2. **测试方便**：适合不想长久用，只想试玩一下的情况。 | 1. **单价最高**：如果你忘了关机，连续跑一个月，费用会比“包年包月”贵很多。 | 临时测试代码、流量波动巨大的业务、短期数据分析任务。         |
| **抢占式实例** (Spot Instance) | **捡漏模式** 购买云厂商闲置的计算资源。           | 1. **价格极低**：最高可省90%，是三种里最便宜的。             | 1. **不可靠**：**随时可能被系统收回**（当别人出价更高或资源紧张时），你的服务器会突然消失。 2. **不支持备案**：**图片中明确提示“抢占式实例不支持备案服务”。** 3. **不适合数据库**：数据容易丢失。 | 仅仅用于学习练手（不存重要数据）、高容错率的运算任务、科学计算。 |

对于流量计费方式，大致如下。【本表格由Gemini Pro生成】

| **特性**       | **按带宽计费 (固定带宽)**                                    | **按流量计费**                                               |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **计费逻辑**   | **包年/包月一口价**。 不管有没有人访问，钱都得交。           | **用多少扣多少**。 按实际流出的数据量（GB）计费。            |
| **带宽速度**   | **通常较慢**。 为了省钱，一般只买 1Mbps - 5Mbps。 人多时网站会卡。 | **非常快**。 通常默认给你开通 100Mbps 的峰值带宽。 瞬间加载能力强。 |
| **费用可控性** | **极高**。 绝不会产生额外费用，预算非常清晰。                | **较低**。 如果网站突然火了，或者被人恶意攻击（刷流量），费用会暴涨。 |
| **适合场景**   | 1. 流量稳定的企业官网。 2. 每天24小时都有人访问的业务。 3. **不想担心流量超标的新手**。 | 1. **个人博客/展示站**（平时没人看，偶尔有人看）。 2. 突发流量大的抢购/活动页。 3. 追求网站打开速度。 |

- 为了备案【阿里云的服务器地点选择在海外，是可以不用备案的，就是访问可能会受到限制等】，笔者选择了包年。

- 为了省钱，笔者选择按流量计费，这里选择了CDT计费模式，每个月的有20GB的免费国内流量，200GB的国外流量，超过则按阶梯计价。[跨产品云上网络带宽统一计费-云数据传输-阿里云](https://help.aliyun.com/zh/cdt/product-overview/what-is-cdt)

  其中，如果国内消耗了50GB流量，国外消耗了120GB流量，则计算如下：国内流量剩余${20-50=-30GB}$，国外流量剩余${200-120=80GB}$，国外流量可以抵消国内流量，即还剩余国外流量${80-30=50GB}$

## 祝贺

到此，就拥有自己的一台服务器了。那服务器有什么用呢？【本表格由Gemini Pro生成】

|                  | **微信小程序后端** | 如果你想开发微信小程序，需要服务器来处理数据、存储用户信息。 | ⭐⭐⭐          |
| **私有云存储**   | **搭建个人网盘**   | **（推荐）** 部署 Nextcloud 或 Cloudreve。相当于拥有了一个自己的“百度网盘”或“Google Drive”，不限速，数据完全掌握在自己手里。 | ⭐⭐           |
|                  | **图床服务**       | 搭建 Cheverlu 等。专门用来存放图片，供你在写 Markdown 文章或发帖时引用。 | ⭐⭐           |
| **开发与运维**   | **代码托管 (Git)** | 部署 GitLab 或 Gitea。相当于搭建一个私人的 GitHub，存放你的私密代码。 | ⭐⭐⭐          |
|                  | **Docker 容器**    | 现代开发的必备。可以在服务器上运行各种容器化应用，环境隔离，互不影响。 | ⭐⭐⭐          |
