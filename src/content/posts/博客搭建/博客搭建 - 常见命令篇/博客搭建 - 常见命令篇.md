## 常见命令

启动开发服务器：`pnmp dev`，可支持实时修改文件，修改后可以立即查看

打包网站：`pnpm build`，生成`dist`文件夹

预览网站：`pnpm preview`，预览`dist`文件夹的内容

## github desktop的commit

参考自`Gemini`回答

| **功能名称**                       | **通俗解释**                                                 |
| ---------------------------------- | ------------------------------------------------------------ |
| **Amend commit (修正提交)**        | **“哎呀，刚才少传了个文件/写错注释了”。** 覆盖最近的一次提交，把新的更改合并进去，不产生新的记录。 |
| **Undo commit (撤销提交)**         | **“刚才这步提交早了，我还没写完”。** 把最近的一次提交“拆解”掉，代码回到未提交（暂存）的状态，就像没提交过一样。 |
| **Reset to commit (重置到此提交)** | **“我要穿越回某天”。** 让整个项目强制回滚到你选中的那个历史时间点。注意：这可能会删除之后的所有工作。 |
| **Checkout commit (检出提交)**     | **“我只想看看那天代码长啥样”。** 切换到那个时间点的只读状态。你可以在这里测试或查看旧代码，但不会改变当前分支的进度。 |
| **Reorder commit (重排提交)**      | **“我想调整提交的先后顺序”。** 比如你先写了功能 B 接着写了功能 A，现在想让 A 排在 B 前面。这通常用于整理干净的提交历史。 |
| **Revert changes (还原更改)**      | **“那次提交的代码有问题，得撤回，但要留下证据”。** 专门创建一个“反向提交”来抵消掉那次提交的影响，历史记录里会看到“撤销某某”的记录。 |
